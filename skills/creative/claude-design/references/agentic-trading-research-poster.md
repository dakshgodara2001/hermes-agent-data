# Agentic trading research poster pattern

Use when the user asks to synthesize research papers/articles about agentic trading, LLM trading agents, financial-agent orchestration, or trading-copilot frameworks into a one-page HTML poster, banner, landing-section, or strategy visual.

## Research sources worth checking first

Search current web/arXiv before relying on this list, but these papers are strong starting anchors:

- TradingAgents: Multi-Agents LLM Financial Trading Framework — trading-firm-inspired role specialization: fundamental, sentiment, technical, bull, bear, trader, and risk agents. Useful product primitive: debate plus specialist lenses before any final decision.
- FinAgent: A Multimodal Foundation Agent for Financial Trading — numerical, textual, visual/K-line inputs; tool augmentation; dual-level reflection; diversified memory retrieval. Useful product primitive: multimodal market-intelligence module plus reflection loop.
- FinMem: Performance-Enhanced LLM Trading Agent with Layered Memory and Character Design — profiling, layered memory, decision-making. Useful product primitive: trader-like memory for recent cues, summary lessons, and behavioral profile.
- FinRobot: Open-Source AI Agent Platform for Financial Applications — financial chain-of-thought, model/task strategy layer, LLMOps/DataOps, foundation-model layer. Useful product primitive: reusable financial-agent platform, not a single chatbot.
- StockBench: Can LLM Agents Trade Stocks Profitably in Real-world Markets? — contamination-free sequential benchmark with prices, fundamentals, news, buy/sell/hold decisions, cumulative return, max drawdown, Sortino. Useful product primitive: benchmark the trading loop, not static QA.
- Orchestration Framework for Financial Agents: From Algorithmic Trading to Agentic Trading — maps algorithmic trading components into planner, orchestrator, data, alpha, risk, portfolio, execution, backtest/evaluation, audit, and memory agents. Useful product primitive: full pipeline as auditable agents with leakage controls.
- Adaptive LLM-based Multi-Agent Systems to Enhance Quantitative Trading Performance / LLM-MAS-DRL — combines LLM semantic analysis, multi-agent collaboration, and DRL/PPO adaptation. Useful product primitive: LLMs interpret unstructured context while deterministic/RL tools handle policy and adaptation.

## Product synthesis pattern for TradePilot-style artifacts

Frame the output as a responsible trading operating system:

- Headline posture: “Not tips. A trading operating system.” / “Decision support, sandbox, and accountability.”
- Avoid: guaranteed-return framing, buy/sell-call bot language, fake performance claims, profit-machine language.
- Show evidence as “reported by papers” and keep claims tied to sources.
- Translate research into product primitives:
  1. Market context agent: Gift Nifty, US/Asia, crude, USD/INR, FII/DII proxy, sector heat.
  2. Stock evidence agents: fundamentals, technicals, sentiment/news, volume/relative strength, event risk.
  3. Bull-vs-bear debate: pro thesis, invalidation, and avoid conditions.
  4. Risk/style gate: separate momentum, swing, and long-term rules; sizing; drawdown guard.
  5. Trader coach/sandbox: user chooses enter / wait / avoid and receives feedback on risk-reward, patience, and invalidation.
  6. Post-market memory: compare suggested setups to actual outcomes; store lessons and behavior patterns, not just P&L.

## Design pattern

For a one-page poster/banner:

- Use a high-density but readable editorial fintech layout.
- Top: thesis + 2–3 sourced impact callouts.
- Middle-left: card grid, one card per paper/research stream.
- Middle-right: “how we build this” pipeline/blueprint.
- Bottom: product wedge, core UX object, moat over time.
- Footer: compliance disclaimer.
- Use sparse dark fintech palette: high-contrast ink, muted gray, green/blue/violet accents.
- Keep source links visible but unobtrusive.

## Verification checklist

- Open the generated HTML locally in a browser.
- Check console for errors.
- Visually inspect desktop layout for clipping, alignment, and readability.
- Confirm every numerical result is either source-attributed or phrased as “reported.”
- Confirm the disclaimer says the artifact is educational/research/journaling/decision-support, not registered advice or guaranteed returns.
